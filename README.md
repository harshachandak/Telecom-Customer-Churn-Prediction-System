# Telecom-Customer-Churn-Prediction-System
This project is a machine learning-based web application designed to predict whether a telecom customer is likely to churn (leave the service) based on key features such as customer tenure, monthly charges, and contract type. The goal of this project is to help businesses identify customers at risk of leaving so that retention strategies can be applied in advance. The model is trained using a Random Forest Classifier on a real-world telecom dataset and provides reliable predictions based on customer behavior patterns.
The project includes complete data preprocessing, exploratory data analysis, feature engineering, model training, and evaluation. Key evaluation metrics such as accuracy, confusion matrix, classification report, and ROC-AUC score are used to measure model performance. The final model is saved and integrated into a Streamlit web application that allows users to input customer details and get real-time churn predictions in an interactive interface.
Technologies used in this project include Python, Pandas, NumPy, Scikit-learn, Matplotlib, Seaborn, and Streamlit. The dataset is cleaned by handling missing values and encoding categorical variables before training the model. Feature importance analysis is also performed to understand the impact of each variable on customer churn.
The project structure is organized into separate modules including a Jupyter notebook for analysis, a preprocessing script, a training script, and a prediction script. The trained model and feature list are saved using pickle and loaded in the Streamlit application for deployment.
To run the project, install the required dependencies using pip install -r requirements.txt and then run the Streamlit app using streamlit run app.py. The application will launch in a browser where users can interact with the model and test different customer scenarios.
